## Purpose

Filter incoming email and Slack to extract only items that require action or carry new knowledge, and append them to `scratch.md` in the metagraph root. Adds Google Calendar context (active meeting at the time of the message) to every entry.

This process is the **capture layer** — it does not classify or create graph nodes. Classification happens in [[P1004]].

---

## Input Sources

| Source   | What to capture                                                                 |
|----------|---------------------------------------------------------------------------------|
| Email    | Starred/flagged messages; messages with detected action items; messages from key contacts defined in personal nodes |
| Slack    | @mentions; threads you participated in; starred/bookmarked messages             |
| Calendar | Active or most recent meeting name at the time of each message (for context)    |

**Do not capture:** newsletters, automated notifications, CC-only messages where no action is implied, messages already processed in a previous run.

---

## Protocol

### Step 1 — Determine time window

Process messages from the last 24 hours (since previous run). Check `## Execution History` for the last successful run timestamp to avoid duplicates.

### Step 2 — Fetch and filter email

Connect to Gmail (or configured email provider):
1. Fetch messages received in the time window.
2. Apply filters — keep only:
   - Starred/flagged messages
   - Messages from senders listed in any `personal` node (subtype: `about`) in `context/personal/`
    - Messages where subject or body contains action signals: "please", "can you", "needed", "deadline", "until [date]", "confirm", "approve", "review", "decisions needed"
3. For each kept message: extract sender, subject, and a 1-sentence summary of the required action or key information.

### Step 3 — Fetch and filter Slack

Connect to Slack workspace:
1. Fetch messages from the time window where you are mentioned (`@vadim`) or have starred.
2. Also fetch threads where you sent a message (you are a participant).
3. For each kept message: extract channel, sender, and a 1-sentence summary.

### Step 4 — Add calendar context

For each captured item, check Google Calendar:
- If a meeting was active within ±15 minutes of the message timestamp → use that meeting name as context.
- If no meeting → use `—`.

### Step 5 — Write to scratch.md

Append entries to `/metagpraph/scratch.md`. Create the file if it doesn't exist.

**Entry format:**
```
YYYY-MM-DD HH:MM | [Meeting name or —] | [email/slack] | WHO → WHAT (max 80 characters)
```

**Recording rules - strictly:**
- Maximum **1 line per email/message**. Never wrap.
- Maximum **80 characters** in the last column.
- Write only **who + action/fact** - do not retell the content.
- Email body, attachments, details - **do not copy to scratch**. If context is needed, the agent will find the email by date + sender during triage.
- If a clear action or fact cannot be extracted from the email in 80 characters - **skip** entry.

**Examples (correct):**
```
2026-04-03 14:32 | [Q2 Planning]    | email | Anton → confirm marketing budget, by Fri
2026-04-03 15:10 | [—]              | slack | Marina → reply about onboarding in #product
2026-04-03 16:45 | [1:1 with team] | email | Investor X → sent term sheet, needs reply
```

**Examples (incorrect - do not do this):**
```
❌ Anton wrote that the Q2 budget has not yet been approved by the CFO...
❌ [long retelling of the email]
```

Do **not** overwrite existing lines. Only append.

### Step 6 — Deduplicate

Before appending, check if a similar entry (same sender + same topic) was already written in the last 48 hours. If yes, skip.

### Step 7 — Log the run

Append a row to **Execution History**:
