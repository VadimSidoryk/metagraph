## Purpose

Scan the entire graph for structural problems that accumulate silently: broken wikilinks, orphaned nodes, INDEX.md inconsistencies, missing required frontmatter fields, and stale nodes. Produces a brief health report and flags issues that require human attention.

This process has no side effects on graph content — it only reads and reports. Fixes are proposed, not auto-applied.

---

## Protocol

### Step 1 — Build the node inventory

Scan all `.md` files in `context/`, `tasks/`, `processes/`, and `archived/` (excluding `scratch.md`, `MENTAL_MODEL.md`, `triage-rules.md`, `README.md`, `CLAUDE.md`, `INDEX.md`).

For each file, extract:
- `id`, `type`, `status`, `modified`, `title` from frontmatter
- All wikilinks in the body: `[[XXXX]]` patterns
- File path and folder location

### Step 2 — Check INDEX.md consistency

Compare the node inventory against INDEX.md:
- Every node file must have a registered entry in INDEX.md
- Every INDEX.md entry must have a corresponding file
- Counters must be ≥ the highest assigned ID in each series

**Flag:** missing registrations, ghost entries (in INDEX.md but no file), counter drift.

### Step 3 — Check wikilink integrity

For every `[[ID]]` found in any file:
- Does a file with that ID exist in the graph?
- Is it in `archived/`? (wikilink to archived node = warn, not error)

**Flag:** broken wikilinks (point to non-existent ID), links to archived nodes.

### Step 4 — Find orphaned nodes

A node is **orphaned** if:
- It has no incoming wikilinks from any other node
- AND it has no outgoing wikilinks
- AND it is not a process (processes are standalone by design)
- AND it is not a decision node (decision logs are standalone by design)

**Flag:** orphaned context nodes and tasks.

### Step 5 — Check required frontmatter

For each node, verify required fields are present and non-empty:
- All nodes: `id`, `type`, `title`, `created`, `modified`, `status`, `assignee`
- Tasks: `priority`, `deadline` (can be null), `creator`
- Goal nodes: `horizon`, `task_refs`
- Knowledge nodes: `source_type`, `source_ref`, `updated_at`
- Personal nodes: `variant`

**Flag:** missing or empty required fields.

### Step 6 — Check stale nodes

- **Tasks** in `todo/` with `modified` older than 30 days → likely forgotten
- **Knowledge nodes** with `updated_at` older than 90 days → may be outdated
- **Goal nodes** whose `horizon` has passed (e.g., Q1-2026 in April 2026) → should be reviewed via P1007

**Flag:** stale tasks, stale knowledge, expired goal horizons.

### Step 7 — Produce health report

Format the report as a concise summary:

```
## Graph Health Report — YYYY-MM-DD

### ✅ OK
- N nodes scanned, M processes, K tasks
- No broken wikilinks
- INDEX.md consistent

### ⚠️ Warnings (review recommended)
- 2 orphaned nodes: [[C1003]] (knowledge, 45d old), [[T1007]] (task, 12d old)
- 1 stale task: [[T1002]] in todo/ since 2026-03-01
- 1 knowledge node not updated in 95d: [[C1005]]

### 🔴 Errors (action required)
- Broken wikilink: [[C9999]] referenced in [[T1003]] — node does not exist
- Missing frontmatter field `priority` in [[T1008]]
- INDEX.md counter drift: last C = C1010 but counter shows C1008
```

### Step 8 — Log the run

Append to **Execution History**:

| Date | Nodes scanned | Errors | Warnings | Notes |
|------|--------------|--------|----------|-------|

### Step 9 — Create fix tasks if needed

For each 🔴 error: create a T-series task in `tasks/todo/` with `priority: high`, `creator: process:P1005`, describing the specific fix needed. Link back to this process in `context_refs`.

Do **not** auto-fix — only create tasks for human/agent review.
