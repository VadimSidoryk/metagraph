## Purpose

Keep all knowledge nodes with remote sources (Jira, Google Docs, Figma, Notion, URL, etc.) up to date by re-fetching their content at least once per day. Ensures the graph reflects current state of external sources.

Applies to nodes where `source_type` is **not** `file` and **not** `human`:
`google_doc` | `jira` | `figma` | `notion` | `url` | `other`

---

## Protocol

### Step 1 — Discover nodes to sync

Scan `context/knowledge/` for all `.md` files. For each file:
- Parse the frontmatter.
- Skip if `source_type` is `file` or `human`.
- Skip if `status` is `archived` or `outdated`.
- Collect into a **sync list**.

### Step 2 — For each node in the sync list

1. Read the current `source_ref` (URL or identifier).
2. Fetch content from the remote source using the appropriate method:

| source_type  | How to fetch                                              |
|--------------|-----------------------------------------------------------|
| `google_doc` | Use Google Docs API or export URL to get latest content   |
| `jira`       | Use Jira REST API (`/rest/api/3/issue/<key>`)             |
| `figma`      | Use Figma API (`/v1/files/<key>`)                         |
| `notion`     | Use Notion API (`/v1/pages/<id>` or `/v1/blocks/<id>`)    |
| `url`        | HTTP GET, extract relevant text content                   |
| `other`      | Best-effort fetch; log method used in the run entry       |

3. Compare fetched content with the current `## Content` section.
4. **If changed:**
   - Update `## Content` with new content (summarize if source is large).
   - Append a line to `## Changelog`: `YYYY-MM-DD: synced from <source_type> — <short description of what changed>`
   - Update `updated_at` and `modified` in frontmatter.
5. **If unchanged:** update only `updated_at` in frontmatter to record the check.
6. If the source is **unreachable** (HTTP error, auth failure, etc.):
   - Do not modify `## Content`.
   - Append to `## Changelog`: `YYYY-MM-DD: sync failed — <error reason>`
   - Set `status: outdated` in frontmatter.
   - Log in the run entry below with status `⚠️ error`.

### Step 3 — Log the run

Append a row to the **Execution History** table in this file:
- Date, nodes found, nodes updated, nodes failed, total duration, any notable issues.

### Step 4 — Recommendations (optional)

If patterns emerge (e.g. a source is repeatedly unreachable, or a node's content never changes), append a recommendation to **Improvement Recommendations**.
