## Purpose

Critically evaluate the progress of all active goals (`subtype: goal`, `status: active`) in the graph. Identify at-risk goals, surface blockers, update Key Results, and produce actionable recommendations. This is **not** a passive sync — the agent should challenge assumptions and flag issues honestly.

---

## Protocol

### Step 1 — Load all active goals

Scan `context/goal/` for files where `status: active`. For each goal, load:
- The goal node itself (objective, key results, horizon, progress notes).
- All linked tasks via `task_refs`: check their current status folder (`todo/`, `in-progress/`, `done/`, `ongoing/`).
- All other `## Related` nodes that are relevant to assessing progress.

### Step 2 — Evaluate each goal critically

For each goal, assess:

| Dimension | Questions to answer |
|---|---|
| **Progress** | What % of Key Results have measurable movement? What is the current value vs target? |
| **Pace** | Given the horizon, is the current pace sufficient to reach targets on time? |
| **Blockers** | Are any linked tasks stalled (`todo` for too long, or `in-progress` without recent `modified`)? |
| **Relevance** | Is the goal still aligned with current decisions and context? Has anything changed that makes it less relevant? |
| **Coverage** | Are there enough active tasks to drive this goal forward, or is it under-resourced? |

### Step 3 — Assign a status signal to each goal

| Signal | Meaning |
|---|---|
| ✅ On track | Pace and progress are sufficient |
| ⚠️ At risk | Progress is lagging or a blocker exists |
| 🔴 Off track | Significant gap; goal unlikely to be achieved without intervention |
| 💀 Stale | No linked task activity in 2+ weeks; goal may be abandoned in practice |

### Step 4 — Update goal nodes

For each goal:
1. Update the `Current` column in the `## Key Results` table with latest known values.
2. Append a line to `## Progress Notes`:
   ```
   YYYY-MM-DD [signal]: <1-2 sentence honest assessment> — <recommended action if any>
   ```
3. Update `modified` in frontmatter.

### Step 5 — Create follow-up tasks if needed

If a goal is ⚠️, 🔴, or 💀:
- Create a new task in `tasks/todo/` describing the specific intervention needed.
- Set `creator: process:P1002` and link back to the goal in `context_refs`.
- Link the new task in the goal's `task_refs` and `## Related`.

### Step 6 — Log the run

Append a row to **Execution History** in this file.
