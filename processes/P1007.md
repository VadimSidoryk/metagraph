## Purpose

Manage the transition between planning horizons: close completed/expired goals with a retrospective, and open the next quarter's goals. Runs at the end of each quarter (last 5 working days).

Without this process, goals accumulate in `active` status forever — there's no ritual of closure, and future quarters have no planning context to build on.

---

## Trigger

Run manually (or via scheduled prompt) during the **last week of the quarter**:
- Q1: last week of March
- Q2: last week of June
- Q3: last week of September
- Q4: last week of December

---

## Protocol

### Phase A — Close the current quarter

#### Step A1 — Load all active goals for the current horizon

Scan `context/goal/` for nodes where `horizon` matches the current quarter (e.g., `Q2-2026`). Load each goal with its linked tasks (via `task_refs`).

#### Step A2 — Run retrospective for each goal

For each goal, assess honestly:

| Question | Answer |
|---|---|
| What % of Key Results were achieved? | — |
| What drove progress? | — |
| What blocked or slowed down? | — |
| Was the goal still relevant by end of quarter? | — |
| What would you do differently? | — |

Write a retrospective note in the goal's `## Progress Notes`:
```
YYYY-MM-DD [RETROSPECTIVE Q2-2026]:
Achieved: X/Y key results (Z%).
Drove: [what worked].
Blocked: [what didn't].
Verdict: achieved ✅ / partial ⚠️ / dropped ❌
Next quarter: [carry forward / close / transform into new goal]
```

**Tone:** focus on learning and momentum, not judgment. What did you learn? What becomes possible now?

#### Step A3 — Update goal status

Based on the retrospective verdict:
- `achieved` → update `status: achieved` in frontmatter
- `dropped` → update `status: dropped`, ask the user for reason (will go in archive record)
- Partial but continuing → leave as active, change `horizon` to next quarter

#### Step A4 — Move completed tasks

All linked tasks in `tasks/done/` — no action needed, they're already done.
Tasks in `tasks/todo/` or `tasks/in-progress/` linked to the closed goal:
- Still relevant next quarter → update their `deadline` and keep
- No longer relevant → propose archiving (ask user)

#### Step A5 — Archive closed goals

For goals with `status: achieved` or `status: dropped`:
1. Ask the user: *"Do you want to archive [[C1001]] — [title]? Reason for closure: [verdict from retrospective]"*
2. On confirmation: move to `archived/context/`, append `## Archive Record`
3. Update INDEX.md

---

### Phase B — Open the next quarter

#### Step B1 — Create next-quarter goals

Ask the user: *"What goals are we setting for [Q3-2026]? Let's create 1-3 goal nodes."*

For each new goal — use standard goal template from CLAUDE.md. Prefill:
- `horizon: Q3-2026`
- `status: active`
- Link any carry-forward tasks from Phase A in `task_refs`
- Connect to relevant knowledge nodes, decisions, personal nodes

#### Step B2 — Create planning tasks

For each new goal, create at least one concrete task in `tasks/todo/` as the first action step. Set `creator: process:P1007`.

#### Step B3 — Log the retrospective run

Append to **Execution History** in this file.
